# Current Context

This document tracks the active development topics, current focus, and immediate next steps for the Vanguard V2.3 project.

## Active Focus

* **Daily Gatekeeper & Engine Sentiment Fix**: Fixed a critical bug in `orchestrator.py` where legacy PyTorch cross-sectional Z-scoring was corrupting the raw features fed to the `daily_xgb` gatekeeper model. Additionally, fixed a baseline drift issue in `model_inference.py` by mean-centering the raw XGBoost `rank:pairwise` margins before calculating `Long_Conviction`, restoring accurate and balanced Engine Sentiment metrics (preventing the 99.4% bearish skew). See [[06. Context & Logs/Conversations/Conv-2026-06-10-Daily-Gatekeeper-Fix|Conversation Log]].
* **V10 Depth-4 Walk-Forward Validation**: Executed a full rolling walk-forward test for the V10+V18 architecture restricted to `max_depth=4`. Discovered that despite potential validation window improvements, the depth-4 configuration does not possess any deployable post-cost walk-forward edge (yielding net `-3.4 bps` Long and `-2.5 bps` Short over the full period at 6bps cost). Confirms that restricting depth does not save the ranking model's edge under strict OOS conditions. See [[06. Context & Logs/Daily Logs/2026-06-10|Daily Log 2026-06-10]].
* **Triple Barrier Method (TBM) Validation**: Designed and executed a comprehensive Triple Barrier Method data labeling and walk-forward validation pipeline, shifting the target from end-of-bar returns to true intra-bar sequential barrier outcomes. ⚠️ **CORRECTED:** the initial SHORT "breakthrough" (+5.18 bps/trade, PF 1.25) was a **cost-sign harness bug** (net cost added instead of subtracted). After the fix, both sides are net-negative post-cost — true SHORT ≈ 50.7% raw / 44.85% net@6bps / +0.63pp skill, LONG dead. **TBM 1h has no deployable post-cost edge.** Always validate RAW vs NET WR per side. See [[06. Context & Logs/Daily Logs/2026-06-09|Daily Log 2026-06-09]] and [[08. Model Analysis/1-Hour Vanguard Model/TBM-1h-Ensemble-Results]].

The engine now prevents trading and unnecessary API calls on market holidays and weekends by checking the date on start and at the beginning of each execution cycle. See [[06. Context & Logs/Daily Logs/2026-06-09|Daily Log 2026-06-09]].

Practical next architecture is a diversified model panel with separate long/short rankers, regime gates, time-of-day specialists, and strict completed-bar walk-forward validation. See [[06.
