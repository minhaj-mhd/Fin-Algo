## 📝 Compacted Session Log
- **Initial Analysis**: The user reported that running the vanguard engine every 15 minutes for 172 tickers is slowing down the system. Initializing investigation into the `scripts/vanguard/` modules to identify the bottleneck.
- **Root Cause Identified**: The `calculate_conviction_scores` loop in `orchestrator.py` sequentially fetched Upstox data for 172 tickers with a 100ms artificial sleep between each fetch, causing severe I/O blocking. Feature generation (`compute_features`) was also executed sequentially.
- **Resolution**: Refactored the data fetch loop and the `compute_features` loop inside `orchestrator.py` to use `concurrent.futures.ThreadPoolExecutor` (max_workers=15 and max_workers=10 respectively).
- **Validation**: Added `test_orchestrator_concurrency.py` in `tests/` which mocks the data fetching block with an artificial delay. Verified that total processing time scales sub-linearly confirming concurrency. Test passed in ~2.11 seconds for 20 tickers.
