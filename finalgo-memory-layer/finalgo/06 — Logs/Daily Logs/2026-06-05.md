# 💬 Conversation Context: SVanguard_TierB_Pure_Short Backtest

- [x] Analyze the backtest logic to see why a 2% loss occurred.
- [x] Fix or improve the strategy to manage risk better or correct its edge.

## 💻 Active Code Files Modified
- [signal_generation.py](file:///c:/Users/loq/Desktop/Trading/finalgo/scripts/vanguard/signal_generation.py)
- [strategy_vanguard_sniper_backtest.py](file:///c:/Users/loq/Desktop/Trading/finalgo/scripts/backtests/strategy_vanguard_sniper_backtest.py)

## 📝 Compacted Session Log
- **Initial Analysis**: Bootstrapping session to investigate SVanguard_TierB_Pure_Short backtest and a recent 2% loss.
- **Step 1**: Discovered `signal_generation.py` was scaling sizes by 5.0 in the generation script, causing duplicate margin application. Fixed to base 1.0/0.75x scales.
- **Step 2**: Created `strategy_vanguard_sniper_backtest.py` via Python generator script and executed it in Jupyter MCP context.
- **Step 3**: Initially saw -214% edge, but realized the backtest script mistakenly evaluated the 1-hour thresholds against the 15-minute model dataset across all times.
- **Step 4**: Corrected backtest to use 1-hour model predictions (`dict_1h`) at 1-hour candle boundaries. First run used `long_conv` and `short_conv` which generated massive trade volume (+10% profit for Tier C, but -16% loss for Tier B).
- **Step 5**: Re-ran the backtest using the raw `long_score` and `short_score`. The trade volume collapsed. I then ran a manual prediction script on the raw CSV data (bypassing the backtest's feature engineering) and perfectly replicated the 818 trades/year mentioned in the research.
- **Decision**: Discovered the ultimate root cause. The live engine (`model_inference.py`) is manually applying a cross-sectional Z-Score transformation `(x - mean) / std` to the features before passing them to the model. However, the training script (`train_ranking_upstox.py`) NEVER applied this transformation; it trained on the raw features. The live engine is feeding distorted feature distributions to the model, causing the margin outputs to squash. We must remove the manual Z-Scoring block from `model_inference.py` to restore the model's edge.
